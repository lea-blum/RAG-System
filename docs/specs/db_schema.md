# תיאור סכמת בסיס הנתונים

מסד הנתונים מנוהל באמצעות **Prisma ORM** ומוגדר כיום לעבוד מול **SQLite**.

## טבלאות קיימות

### Appointment (`appointments`)
מייצגת תור בודד במערכת.

- **id** (`Int`, Primary Key, Auto-increment)  
  מזהה ייחודי לכל תור.

- **scheduledAt** (`DateTime`)  
  הזמן המתוכנן של התור (תאריך ושעה).

- **treatmentType** (`String`)  
  סוג הטיפול המתוכנן (לדוגמה: "בדיקה כללית", "טיפול שיניים", "מעקב").

- **status** (`AppointmentStatus` enum)  
  מצב התור:
  - `SCHEDULED` – תור שנקבע ועדיין לא בוצע.
  - `COMPLETED` – תור שהסתיים בהצלחה.
  - `CANCELLED` – תור שבוטל.

## הערות פרטיות
- **נכון לעכשיו, המערכת אינה שומרת היסטוריית תורים שנמחקו**, מטעמי פרטיות והקטנת שמירת מידע אישי לאורך זמן.
- מדיניות זו עשויה להשתנות בעתיד, ובמקרה כזה תתבצע התאמה לסכמת הנתונים ותעודכן החלטה מתאימה ב־`docs/decisions/`.
