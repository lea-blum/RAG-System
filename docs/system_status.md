# מצב מערכת – מערכת לניהול תורים למרפאה

## תקציר נוכחי
- **סטטוס**: שלב הקמה ראשוני (Skeleton).
- **Frontend**: הוגדר פרויקט Next.js בסיסי עם תיקיית `app`, כולל Layout ומסך בית ראשוני בעיצוב כחול רפואי.
- **Database / ORM**: הוגדר Prisma עם ספק SQLite, ללא מודלים עסקיים מפורטים עדיין (תורים, רופאים, מטופלים).
- **תיעוד**: קיימים קבצי דרישות התחלתיים, החלטה ארכיטקטונית ראשונה ויומן פעילות.

## צעדים הבאים המומלצים
- הגדרת מודלים ב-Prisma עבור רופאים, מטופלים ותורים.
- יצירת API Routes לניהול תורים ורופאים.
- הרחבת ה-UI למסכים ייעודיים לזימון תורים ולניהול רופאים.
