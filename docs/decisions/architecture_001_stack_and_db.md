# החלטה ארכיטקטונית 001 – סטאק וטכנולוגיות בסיס

## הקשר
הקמת מערכת לניהול תורים למרפאה כפרויקט Fullstack חדש.

## החלטות
- **Frontend**: שימוש ב-`Next.js` (גרסה 15.3) עם מבנה `app` router.
- **Backend / Server Rendering**: שימוש ביכולות ה-Fullstack של Next.js (Server Components, API Routes בהמשך).
- **ORM ושכבת גישה לנתונים**: שימוש ב-`Prisma ORM` (גרסה 7.4.1) לצורך דגם הנתונים ושכבת הגישה למסד הנתונים.
- **מסד נתונים**: בשלב זה נבחר **SQLite** כספק ברירת מחדל (דרך `prisma init --datasource-provider sqlite`) לצורך פיתוח מהיר וללא תלות בשרת חיצוני.

## נימוקים
- Next.js מספק שילוב נוח בין Server ו-Client ומאפשר התפתחות הדרגתית לפונקציונליות מתקדמת (API Routes, Middleware וכו').
- Prisma מאפשר דגם סכימה טיפוסי, הגירה פשוטה ו-Type Safety גבוה בצד השרת.
- SQLite מתאים לשלב ההתחלתי של המערכת, במיוחד בפיתוח מקומי. בעתיד ניתן להחליף לספק אחר (PostgreSQL/MySQL) תוך התאמות מינימליות בסכימה וההגדרות.

## השלכות
- כל מודול חדש במערכת (כגון תורים, רופאים, מטופלים) יוגדר תחילה במודל Prisma.
- בעת מעבר לסביבת Production, תתקבל החלטה חדשה לגבי סוג מסד הנתונים, ותתועד כהחלטה ארכיטקטונית נפרדת.
